---
title: 'List Folders'
description: 'Retrieve a list of all folders for the authenticated user'
---

Retrieve all folders belonging to the authenticated user. You can filter by parent folder or get only root-level folders.

## Endpoint

```http
GET /api/user/folders
```

## Authentication

Requires authentication via API token in the `Authorization` header.

## Query Parameters

<ParamField query="noincl" type="boolean" default="false">
  When `true`, excludes files from the response. Only folder metadata and counts will be returned.
</ParamField>

<ParamField query="user" type="string">
  User ID to filter folders by. Only works for admin users when querying other users' folders.
</ParamField>

<ParamField query="parentId" type="string">
  Filter folders by parent folder ID. Returns only folders that are children of the specified parent.
</ParamField>

<ParamField query="root" type="boolean" default="false">
  When `true`, returns only root-level folders (folders without a parent).
</ParamField>

## Response

Returns an array of folder objects ordered by creation date (newest first).

<ResponseField name="id" type="string" required>
  Unique identifier for the folder
</ResponseField>

<ResponseField name="name" type="string" required>
  The folder name
</ResponseField>

<ResponseField name="public" type="boolean" required>
  Whether the folder is publicly accessible
</ResponseField>

<ResponseField name="allowUploads" type="boolean" required>
  Whether uploads are allowed to this folder
</ResponseField>

<ResponseField name="createdAt" type="string" required>
  ISO 8601 timestamp of when the folder was created
</ResponseField>

<ResponseField name="updatedAt" type="string" required>
  ISO 8601 timestamp of when the folder was last updated
</ResponseField>

<ResponseField name="userId" type="string" required>
  ID of the user who owns this folder
</ResponseField>

<ResponseField name="parentId" type="string | null">
  ID of the parent folder, or `null` if this is a root folder
</ResponseField>

<ResponseField name="files" type="array">
  Array of file objects in this folder. Only included when `noincl` is not set.
  
  <Expandable title="File object properties">
    <ResponseField name="id" type="string">
      File ID
    </ResponseField>
    <ResponseField name="name" type="string">
      File name on the server
    </ResponseField>
    <ResponseField name="originalName" type="string | null">
      Original filename when uploaded
    </ResponseField>
    <ResponseField name="type" type="string">
      MIME type of the file
    </ResponseField>
    <ResponseField name="size" type="number">
      File size in bytes
    </ResponseField>
    <ResponseField name="views" type="number">
      Number of times the file has been viewed
    </ResponseField>
    <ResponseField name="maxViews" type="number | null">
      Maximum views before deletion
    </ResponseField>
    <ResponseField name="favorite" type="boolean">
      Whether the file is marked as favorite
    </ResponseField>
    <ResponseField name="password" type="boolean">
      Whether the file is password protected
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="_count" type="object">
  Counts of related entities
  
  <Expandable title="Count properties">
    <ResponseField name="children" type="number">
      Number of child folders
    </ResponseField>
    <ResponseField name="files" type="number">
      Number of files in this folder
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="parent" type="object | null">
  Parent folder information, or `null` if this is a root folder
  
  <Expandable title="Parent properties">
    <ResponseField name="id" type="string">
      Parent folder ID
    </ResponseField>
    <ResponseField name="name" type="string">
      Parent folder name
    </ResponseField>
    <ResponseField name="parentId" type="string | null">
      ID of the parent's parent folder
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

```bash cURL
curl -X GET "https://your-zipline-instance.com/api/user/folders" \
  -H "Authorization: your-api-token"
```

```bash Get root folders only
curl -X GET "https://your-zipline-instance.com/api/user/folders?root=true" \
  -H "Authorization: your-api-token"
```

```bash Get folders without files
curl -X GET "https://your-zipline-instance.com/api/user/folders?noincl=true" \
  -H "Authorization: your-api-token"
```

```bash Get children of a specific folder
curl -X GET "https://your-zipline-instance.com/api/user/folders?parentId=abc123" \
  -H "Authorization: your-api-token"
```
